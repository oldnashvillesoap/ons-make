# ArtisanOS - Artisan Production Management System

A web-based application for small-batch producers to manage recipes, inventory with FIFO costing, and production planning.

## Features

✨ **Recipe Management**
- Create and store product recipes with ingredient lists
- Automatic cost calculation based on current inventory prices
- Track cost per unit for informed pricing decisions

📦 **Inventory Management**
- Log material receipts with cost tracking
- Support for multiple units: gallons, pounds, ounces, pieces
- FIFO (First-In, First-Out) costing for realistic cost accounting
- Visual inventory summary and detailed ledger

🏭 **Production Batches**
- Plan batches in advance with automatic material reservation
- Execute batches with permanent FIFO material deduction
- Track batch costs and unit economics
- Optional custom ingredient adjustments per batch

⚠️ **Forecasting & Alerts**
- Identify material shortages from planned production
- Prevent production delays and rush purchases
- Material need projection

📚 **Comprehensive Documentation**
- In-app Help tab with detailed workflows
- Best practices for production planning
- Unit conversion reference

## Getting Started

### Prerequisites
- A modern web browser (Chrome, Firefox, Safari, Edge)
- Firebase account with Firestore database enabled

### Initial Setup

1. **Open the Application**
   - Open `index.html` in your web browser
   - The app runs entirely in your browser; no server installation needed

2. **Create Your Firebase Project** (if not already done)
   - Go to [Firebase Console](https://console.firebase.google.com)
   - Create a new project (or use existing)
   - Enable Authentication (Email/Password method)
   - Enable Firestore Database
   - Get your Firebase config and update in `index.html` (lines 43-50)

3. **Create a User**
   - Use the login screen to sign up with an email and password
   - Firebase will automatically create your user account

### Quick Start Workflow

1. **Create a Recipe**
   - Go to "Recipes" tab
   - Fill in: Name, Category, Yield (number of units)
   - Add ingredients in format: "Ingredient Name - 50g" (one per line)
   - Click "Save Recipe"

2. **Log Inventory**
   - Go to "Inventory" tab
   - Enter material details: Material Name, Unit, Quantity, Total Cost Paid
   - Click "Log to FIFO Ledger"
   - View inventory summary and ledger

3. **Plan a Production Batch**
   - Go to "Batches" tab
   - Click "Plan New Batch"
   - Select recipe, production date, optional notes
   - Optionally override ingredient quantities
   - Click "Save as Planned"
   - System checks forecasting; alerts show material shortages

4. **Execute a Batch**
   - When ready to produce, click "Execute Batch"
   - System:
     - Consumes materials using FIFO (oldest first)
     - Locks in costs
     - Updates inventory
     - Calculates total and per-unit cost

5. **Monitor & Adjust**
   - Review recipe costs (update automatically as prices change)
   - Check inventory summary for stock levels
   - Use forecasting alerts to plan purchases

## Key Concepts Explained

### FIFO (First-In, First-Out) Costing
When you use materials in production, the system consumes the oldest inventory first. This ensures:
- Realistic cost tracking aligned with how materials are actually used
- Automatic cost-per-unit adjustment as material prices fluctuate
- No expired materials sitting in the back

**Example:**
- You buy 10 lbs Lye for $40 (March) → $4/lb cost
- You buy 10 lbs Lye for $50 (April) → $5/lb cost
- When executing a batch needing 15 lbs:
  - 10 lbs from March batch @ $4/lb = $40
  - 5 lbs from April batch @ $5/lb = $25
  - **Total: $65** (not $67.50 if evenly averaged)

### Unit Conversions
Recipes specify precise amounts (grams, pieces). Inventory tracks bulk units (gallons, pounds).

The system automatically converts:
- 1 gallon = 3,785.41 grams
- 1 pound = 453.592 grams
- 1 ounce = 28.3495 grams

### Batch Lifecycle
- **Planned**: Created, materials reserved, not yet consumed
- **Executed**: Materials permanently deducted from inventory, costs locked

## Common Workflows

### Setting Up a New Product Line

1. Create a base recipe with standard ingredients
2. Plan test batches with custom ingredients
3. Log actual costs from suppliers
4. Watch recipe costs stabilize and guide pricing

### Managing Seasonal Price Changes

1. When material prices change, log new receipt at new price
2. Existing inventory stays at original cost
3. New batches automatically use blend of old (cheap) and new (expensive) materials
4. See realistic impact on product cost

### Forecasting Production Needs

1. Plan next month's production as batches
2. Review forecasting alerts
3. Order materials 2-4 weeks ahead
4. Avoid rush purchases and production delays

## Tips & Best Practices

✅ **DO:**
- Be consistent with ingredient names (case matters for matching)
- Log inventory receipts promptly for accurate costing
- Plan batches in advance to identify shortages early
- Use notes to track experimental batches
- Review the Help tab for detailed workflows

❌ **DON'T:**
- Change ingredient names inconsistently ("Goat Milk" vs "goat milk")
- Execute batches without reviewing forecasting alerts
- Log duplicate inventory receipts
- Expect immediate database sync (slight delays are normal)

## Troubleshooting

### Application won't load
- Ensure JavaScript is enabled in your browser
- Check browser console for errors (F12 → Console tab)
- Clear browser cache and reload

### Can't log in
- Verify email and password are correct
- Check Firebase project is properly configured in `index.html`
- Ensure Authentication is enabled in Firebase Console

### Recipe costs show $0
- Ingredient names must **exactly** match inventory material names
- Example: "Goat Milk" in recipe must match "Goat Milk" in inventory (case-sensitive)

### Can't execute batch
- "Insufficient [material]" means you don't have enough in inventory
- Log more material or reduce custom ingredient quantities

### Data not showing up
- Check that you're logged in
- Firestore queries filter by user ID for data isolation
- Reload the page if data hasn't synced from Firebase

## Deployment

### Local Testing
- Simply open `index.html` in your browser
- Use Firebase emulator for offline testing (optional)

### Production Deployment
- Host on Firebase Hosting (recommended)
- Or any static hosting (GitHub Pages, Netlify, etc.)
- No backend server needed

**To deploy to Firebase Hosting:**
```bash
npm install -g firebase-tools
firebase login
firebase init hosting
# Copy index.html to public/ folder
firebase deploy
```
